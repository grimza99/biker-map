# Notion 문서화 가이드

<strong>버전 : </strong> v1

<strong>생성 날짜 : </strong> 2026-05-21

<strong>최신 업데이트 날짜 : </strong> 2026-05-21

이 문서는 Notion 업무일지, 위키, 중간 현황, 리팩토링 문서 생성 시 참고합니다.

## 기준 위치

프로젝트 위키는 Notion의 `biker-map` 위키입니다.

- 문서 생성 후 ancestor path를 확인합니다.
- 자동화나 MCP 연결이 다른 base URL을 바라볼 수 있으므로 생성 위치를 검증합니다.
- title property가 비어 `새 페이지`로 보이면 title property를 다시 update합니다.
- 업무일지 보고 페이지는 title 의 형식을 `yyyy-mm-dd 업무일지` 로 합니다.

## 문서 종류

- 업무일지: 날짜별 작업 내용, 결정, 후속 작업
- 중간 현황: MVP 구현 상태, MVP 이후 backlog
- 리팩토링 후보: 우선순위, 대상 파일, 이유, PR 분리 전략
- 앱 계약 문서: 앱 개발에 영향을 주는 API/auth/notification 변경

## 작성 원칙

- 단순 기록보다 이후 의사결정에 필요한 맥락을 남깁니다.
- 구현 완료와 보류 사항을 분리합니다.
- 코드 파일이나 PR이 근거라면 대상 경로와 이유를 함께 적습니다.
- 문서 생성 후 링크를 사용자에게 공유합니다.
- 사용자와 에이전트의 맥락 동기화를 위한 문서임으로 줄바꿈, 들여쓰기, 구분선을 충분히 활용하여사용자의 가독성을 최우선으로 한다.
